## ✨ Возможности

- Скачивание видео, фото и аудио с множества платформ социальных сетей
- Интерфейс Telegram-бота для удобного взаимодействия
- Фоновая обработка задач с помощью Taskiq
- Кэширование Redis для повышения производительности
- База данных PostgreSQL для истории пользователей и хранения данных
- Поддержка нескольких языков (английский, русский)
- Выбор разрешения видео
- Аутентификация на основе cookies для доступа к платформам

## 🌐 Поддерживаемые платформы

- YouTube (видео и Shorts)
- Instagram (API и yt-dlp)
- TikTok (API и yt-dlp)
- X (Twitter)
- VK (видео и клипы)
- Rutube
- Pinterest
- Dzen
- M3U8 потоки

## 📦 Требования

- Python 3.10+
- [Poetry](https://python-poetry.org/) для управления зависимостями
- [Task](https://taskfile.dev/) для автоматизации задач
- Docker и Docker Compose для развертывания
- PostgreSQL 15+

## 🚀 Установка

1. **Клонируйте репозиторий:**

```bash
git clone <repository-url>
cd saver_backend
```

2. **Установите зависимости:**

```bash
poetry install
```

3. **Установите pre-commit хуки:**

```bash
poetry run pre-commit install
```

Pre-commit хуки автоматически запускаются перед каждым коммитом и проверяют:
- Форматирование кода с помощью **Black**
- Качество кода с помощью **Ruff**
- Проверку типов с помощью **MyPy**

4. **Настройте переменные окружения:**

Скопируйте пример файла окружения и настройте его:

```bash
cp .env.example .env
```

Затем отредактируйте `.env` с вашими реальными значениями. См. раздел [Переменные окружения](#-переменные-окружения) для подробных описаний.

## ⚙️ Конфигурация

### Настройка Cookies

Некоторые платформы требуют cookies для аутентификации. Разместите файлы cookies в директории `cookies/`:

- `cookies/instagram_ydl/` - Cookies для Instagram
- `cookies/vk_clips_ydl/` - Cookies для клипов VK
- `cookies/vk_video_ydl/` - Cookies для видео VK
- `cookies/youtube_shorts_ydl/` - Cookies для YouTube Shorts
- `cookies/youtube_video_ydl/` - Cookies для видео YouTube

### Миграции базы данных

Запустите миграции базы данных:

```bash
task migrate
```

Или вручную с помощью alembic:

```bash
poetry run alembic upgrade head
```

## 💻 Разработка

### Локальный запуск

1. **Соберите локальное окружение (только первый раз):**

```bash
task build-local
```

2. **Запустите локальное окружение:**

```bash
task deploy-local
```

Это запустит все необходимые сервисы в Docker контейнерах.

### Доступные команды Task

Проект использует [Taskfile](https://taskfile.dev/) для автоматизации задач. Доступные команды:

- `task deploy` - Сборка и развертывание в продакшен
- `task migrate` - Запуск миграций базы данных
- `task build-local` - Сборка локального окружения разработки
- `task deploy-local` - Запуск локального окружения разработки
- `task locales` - Извлечение и компиляция файлов переводов

Просмотр всех доступных задач:

```bash
task --list
```

### Локализация

Для работы с переводами:

```bash
task locales
```

Эта команда:
1. Извлечет все переводимые строки из кодовой базы
2. Обновит существующие файлы переводов
3. Скомпилирует файлы `.po` в формат `.mo`

Файлы переводов находятся в директории `locales/`.

## 🚢 Развертывание

### Продакшен развертывание

1. **Развертывание в продакшен:**

```bash
task deploy
```

Эта команда:
- Соберет Docker образы
- Проверит, запущены ли уже контейнеры
- Если запущены: выполнит плавное обновление без простоя с помощью `docker rollout`
- Если не запущены: запустит все сервисы с помощью `docker-compose up`
- Запустит миграции базы данных
- Перезагрузит конфигурацию nginx

### Сервисы Docker Compose

Приложение состоит из нескольких сервисов:
- **api** - Веб-приложение FastAPI
- **taskiq-worker** - Обработчик фоновых задач
- **taskiq-scheduler** - Планировщик задач
- **migrator** - Запуск миграций базы данных
- **nginx** - Обратный прокси-сервер
- **postgres** - База данных PostgreSQL
- **redis** - Кэш Redis

### Ключевые компоненты архитектуры

#### 1. Источники загрузки (`saver_backend/services/downloaders/`)

Все источники загрузки реализованы на основе абстрактного класса `BaseSourceController`. Каждая платформа имеет свой контроллер, который расширяет этот базовый класс.

**Базовый контроллер** (`base_source.py`):
- Предоставляет общий интерфейс для всех источников загрузки
- Обрабатывает кэширование, отслеживание истории и управление пользователями
- Управляет жизненным циклом загрузки и обработкой ошибок

**Resolver** (`resolver.py`):
- Определяет платформу-источник по URL
- Направляет запросы к соответствующим контроллерам
- Обрабатывает сопоставление и валидацию URL-шаблонов

**Контроллеры платформ**:
- Каждая платформа имеет выделенный контроллер (например, `youtube_video_ydl_source.py`)
- Реализует специфичную для платформы логику загрузки
- Обрабатывает аутентификацию и API-вызовы платформы

#### 2. Telegram бот (`telegram_bot/`)

Точка входа для всех взаимодействий с Telegram:
- **handlers/** - Обрабатывает сообщения пользователей, команды и callback'и
- **middlewares/** - Конвейер обработки запросов (авторизация, i18n и т.д.)
- **keyboards/** - Интерактивные клавиатуры для пользовательского интерфейса
- **filters/** - Пользовательские фильтры сообщений

#### 3. Менеджер задач (`task_manager/`)

Управляет всей фоновой обработкой задач:
- **tasks.py** - Определяет асинхронные задачи для загрузки медиа
- Все взаимодействия с контроллерами загрузки происходят здесь
- Обрабатывает планирование и выполнение задач
- Предоставляет внедрение зависимостей для задач

#### 4. Web API (`web/`)

Приложение FastAPI для обработки webhook и REST API:
- **api/** - REST эндпоинты для внешних интеграций
- **webhook/** - Приемники webhook Telegram

## 🔧 Качество кода

Проект использует несколько инструментов для поддержания качества кода:

- **Black** - Форматирование кода
- **Ruff** - Быстрый Python линтер
- **MyPy** - Статическая проверка типов
- **Pre-commit** - Git хуки для автоматических проверок

Эти инструменты запускаются автоматически при каждом коммите через pre-commit хуки.

Для запуска проверок вручную:

```bash
# Форматирование кода
poetry run black saver_backend tests

# Проверка кода линтером
poetry run ruff check saver_backend tests --fix

# Проверка типов
poetry run mypy saver_backend
```

## 📝 Переменные окружения

Все переменные окружения имеют префикс `SAVER_BACKEND_`. Полный файл `.env.example` предоставлен в репозитории.

### Настройки приложения

| Переменная | Описание | По умолчанию |
|-----------|----------|--------------|
| `SAVER_BACKEND_HOST` | Адрес хоста приложения | `127.0.0.1` |
| `SAVER_BACKEND_PORT` | Порт приложения | `8000` |
| `SAVER_BACKEND_WORKERS_COUNT` | Количество воркеров Uvicorn | `1` |
| `SAVER_BACKEND_RELOAD` | Включить горячую перезагрузку (только dev) | `False` |
| `SAVER_BACKEND_ENVIRONMENT` | Имя окружения (local/dev/prod) | `local` |
| `SAVER_BACKEND_LOG_LEVEL` | Уровень логирования | `INFO` |

### Конфигурация базы данных

| Переменная | Описание | По умолчанию |
|-----------|----------|--------------|
| `SAVER_BACKEND_DB_HOST` | Хост PostgreSQL | `localhost` |
| `SAVER_BACKEND_DB_PORT` | Порт PostgreSQL | `5432` |
| `SAVER_BACKEND_DB_USER` | Пользователь БД | `saver_backend` |
| `SAVER_BACKEND_DB_PASS` | Пароль БД | `saver_backend` |
| `SAVER_BACKEND_DB_BASE` | Имя базы данных | `saver_backend` |
